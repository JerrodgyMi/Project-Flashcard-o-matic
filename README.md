# Post-Submission Reflection

## 1. Problem-solving approach
I started by reading the project requirements and breaking them into routes and components. I then implemented routing step-by-step, testing each route as I built it. When errors occurred, I used the test output and browser console to trace missing imports and routing issues.

## 2. Key technical decision
I chose to centralize routing inside `App.js` and keep `index.js` minimal with only the BrowserRouter wrapper. This helped avoid router nesting issues and made the routing structure easier to debug.

## 3. AI use disclosure
No AI was used.

## 4. Area to improve
I want to improve my understanding of React Router v6 and data flow between components, especially how props and API calls interact with routing and state management.
